YML - базовые инструкции для серверов, активен когда в JSON нету настроек для сервера
JSON - индвидуальные настройки для каждого сервера

Цель: обеспечить уменьшение кол-во строчек в базах данных JSON
За счет того, что базовые настройки находятся в yml файлах, и их не надо бесконечно дублировать для каждого сервера по отдельности в JSON
Если будут настройки, которые не похожи на базовые, то в JSON заносятся индивидуальные данные
